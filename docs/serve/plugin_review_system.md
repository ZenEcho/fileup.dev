# 插件评价闭环系统（M4）

本文档定义插件市场评价系统的后端耦合实现，基于 `plugins.controller.ts` + `plugins.service.ts` + Prisma schema 当前代码。

## 1. 模块目标

评价系统用于插件详情页反馈闭环，当前包含：

- 登录用户发布评价（评分 + 文本）。
- 插件作者或管理员对评价追加回复（多条）。
- 管理员删除评价。
- 评价人和回复人支持“作者/管理员”身份标识。

## 2. 权限模型

| 操作 | 未登录 | 登录用户 | 插件作者 | 管理员 |
| --- | --- | --- | --- | --- |
| 查看评价列表 | ✅ | ✅ | ✅ | ✅ |
| 发布评价 | ❌ | ✅ | ✅ | ✅ |
| 回复评价 | ❌ | ❌ | ✅（仅自己插件） | ✅ |
| 删除评价 | ❌ | ❌ | ❌ | ✅ |

补充约束：

- `POST /reviews` 需要插件可发布（公开 + 至少 1 个 APPROVED 版本）。
- 回复接口允许在私有插件场景使用（用于作者/管理员内部处理）。

## 3. 数据模型

## 3.1 PluginReview

| 字段 | 说明 |
| --- | --- |
| `id` | 评价 ID |
| `pluginId` / `userId` | 所属插件与评价人 |
| `rating` | 1~5 整数 |
| `content` | 评价正文（<= 2000） |
| `authorReply*` | 旧版单回复兼容字段（保留） |
| `createdAt` / `updatedAt` | 时间戳 |

## 3.2 PluginReviewReply

| 字段 | 说明 |
| --- | --- |
| `id` | 回复 ID |
| `reviewId` / `userId` | 目标评价与回复人 |
| `content` | 回复内容（<= 2000） |
| `createdAt` / `updatedAt` | 时间戳 |

说明：

- 当前实现已以 `PluginReviewReply` 作为主回复模型。
- 返回层仍保留 `authorReply` 字段，避免旧前端结构断裂。

## 4. 接口契约

统一前缀：`/api/plugins`

## 4.1 获取评价列表

`GET /:id/reviews`

返回结构：

- `summary.total`
- `summary.averageRating`（保留 2 位小数）
- `reviews[]`
- `reviews[].replies[]`（按时间升序）
- `reviews[].authorReply`（兼容字段，取最后一条回复）

## 4.2 发布评价

`POST /:id/reviews`

请求体：

```json
{
  "rating": 5,
  "content": "评价内容"
}
```

校验规则：

- `rating` 必须是 `1~5` 整数。
- `content` 去空白后不能为空且 `<= 2000`。
- 同一用户对同一插件已评价时，当前实现返回错误（不做 upsert）。

## 4.3 回复评价

`PATCH /:id/reviews/:reviewId/reply`

请求体：

```json
{
  "content": "回复内容"
}
```

校验规则：

- 仅插件作者或管理员可调用。
- 内容去空白后不能为空且 `<= 2000`。
- 每次调用都会新增一条 `PluginReviewReply` 记录。

## 4.4 删除评价

`DELETE /:id/reviews/:reviewId`

规则：

- 仅管理员可调用（控制器层直接拦截）。

## 5. 前端耦合点

## 5.1 市场页（`PluginMarketplace.vue`）

- 打开插件详情时拉取 `GET /plugins/:id/reviews`。
- 已登录用户可发评价；若已评价，前端直接禁用提交并提示。
- 作者/管理员可逐条回复。
- 删除按钮仅管理员显示。

## 5.2 管理后台（`AdminReview.vue`）

- 评价管理模块复用同一组接口。
- 管理员可统一查看、删除、回复。
- “垃圾评价”标记仅前端本地状态，不写后端。

## 6. 返回结构兼容策略

后端当前输出同时包含：

- 新结构：`replies[]`
- 兼容字段：`authorReply`

目的：

- 新旧页面都能消费同一返回。
- 逐步迁移期间避免一次性破坏调用方。

## 7. 边界与风险

1. 评价读写大量使用 `$queryRaw/$executeRaw`，维护成本高于 Prisma ORM。
2. 服务端已限制“同用户同插件重复评价”，但数据库层当前不是强唯一约束。
3. 插件未公开或无 APPROVED 版本时，发布评价会被拒绝。
4. 回复权限依赖“插件作者或管理员”判定，调用方应明确显示权限提示。

## 8. 验收清单

- [ ] 未登录用户不能发布评价。
- [ ] 已登录用户可发布一次评价，重复提交被拦截。
- [ ] 作者/管理员可追加多条回复并按时间展示。
- [ ] 非管理员不能删除评价。
- [ ] 市场页与后台页都能正确渲染 `replies[]` 与 `authorReply` 兼容字段。
